<p><b>Dash User Guide</b></p>
<p></p>
<p><span style="font-weight: 400;"> &nbsp;&nbsp;&nbsp;&nbsp;Dash was designed for online marketing agencies, with the express purpose of benefiting their customers. The software provides all features for a customer of an online marketing agency to make informed choices on their investment into campaigns lead by an agency. This is made possible through a plethora of features, allowing anyone, ranging from those new to online advertising, to seasoned veterans versed in data analysis, to support all investment choices with unbiased data.</span></p>
<p></p>
<p><b>PREREQUISITES</b></p>
<p><span style="font-weight: 400;">Dash is based on the Java Framework. It cannot be run on machines not supporting the JVM.</span></p>
<p><span style="font-weight: 400;">Dash expects users to acquire log files of campaigns from their agencies. These must be stored on the same machine as Dash, preferably in Dash&rsquo;s directory, but otherwise may be stored wherever the user wishes. </span></p>
<p><b>1 STARTING THE APPLICATION</b></p>
<p><b>1.1 Opening Dash and the Start Screen</b></p>
<ol>
<li style="font-weight: 400;"><span style="font-weight: 400;">Firstly, ensure the Dash application is installed. </span></li>
<li style="font-weight: 400;"><span style="font-weight: 400;">Once installed, open Dash. </span></li>
<li style="font-weight: 400;"><span style="font-weight: 400;">When Dash has opened, load your campaign (see </span><b>3.0</b><span style="font-weight: 400;">). </span></li>
<li style="font-weight: 400;"><span style="font-weight: 400;">Once you have successfully loaded your campaign, Dash will enter the main application window.</span></li>
<li style="font-weight: 400;"><span style="font-weight: 400;">Viewing metrics, graphs and comparing campaigns are features now available to you. For more information on any of these, please see sections </span><b>3.1</b><span style="font-weight: 400;"> through </span><b>3.3</b><span style="font-weight: 400;">. For advanced setting management, see </span><b>3.4</b><span style="font-weight: 400;">.</span></li>
</ol>
<p><b>1.2 The Main Application Window</b></p>
<p><span style="font-weight: 400;"> &nbsp;&nbsp;&nbsp;&nbsp;After you have completed </span><b>1.1</b><span style="font-weight: 400;">, you will now be in Dash&rsquo;s main window. Here, all features are listed for you. These include features </span><b>3.1</b><span style="font-weight: 400;"> through </span><b>3.4</b><span style="font-weight: 400;"> of this guide, as well as </span><b>2.2</b><span style="font-weight: 400;"> for application closure. </span></p>
<p><span style="font-weight: 400;"> &nbsp;&nbsp;&nbsp;&nbsp;The main feature of this window is the overview capabality. Hovering your mouse over the button representative of each feature gives an overview panel of this sections functionality in the main display panel. The contents of each overview is explained in sections </span><b>3.1</b><span style="font-weight: 400;"> through </span><b>3.4 </b><span style="font-weight: 400;">in detail.</span></p>
<p><b>2 CLOSING THE APPLICATION</b></p>
<p><b>2.1 Closing Dash from the Start Screen</b></p>
<p><span style="font-weight: 400;"> &nbsp;&nbsp;&nbsp;When in the start screen, Dash has not yet loaded any campaign files. Whether you opt to use the Exit button provided, or simply close the application window by clicking close at the top, Dash will safely close. </span></p>
<p><b>2.2 Closing Dash from the Main Application Window</b></p>
<p><span style="font-weight: 400;"> &nbsp;&nbsp;&nbsp;Dash is designed, with campaign data loaded, to safely wipe all data logs from the application before closing. This is true whether you use the Exit button provided, or simply close the window at the top.</span></p>
<p><b>3 USING THE APPLICATION</b></p>
<p><b>3.0 Loading Campaign Files</b></p>
<p><b>3.0.1 Loading Files in Dash&rsquo;s Directory</b></p>
<p><span style="font-weight: 400;"> &nbsp;&nbsp;&nbsp;&nbsp;Whenever presented with the option to load campaign data, Dash will always check it&rsquo;s own directory first if no campaigns have been loaded. Once an initial campaign is found in Dash&rsquo;s directory, it will instantly be loaded in for you. If a campaign was already loaded, see </span><b>3.0.2</b><span style="font-weight: 400;">.</span></p>
<p><b>3.0.2 Loading Files Outside Dash&rsquo;s Directory</b></p>
<p><span style="font-weight: 400;"> &nbsp;&nbsp;&nbsp;&nbsp;When loading additional campaigns into Dash, or if an initial campaign can not be found by Dash in it&rsquo;s directory, a file navigation window will appear:</span></p>
<p></p>
<ol>
<li style="font-weight: 400;"><span style="font-weight: 400;">In the file navigator, select the directory where you have stored your files</span></li>
<li style="font-weight: 400;"><span style="font-weight: 400;">Once in said directory, select the campaign folder with all files to be loaded into Dash.</span></li>
<li style="font-weight: 400;"><span style="font-weight: 400;">Click on the Load Campaign button</span></li>
</ol>
<p><span style="font-weight: 400;"> &nbsp;&nbsp;&nbsp;Dash will then load your campaign. In the event of a failure to do so, Dash will inform you and offer the choice to try again or cancel the action. </span></p>
<p><b>3.1 Metrics</b></p>
<p><b>3.1.1 Overview of Metrics </b></p>
<p><span style="font-weight: 400;"> &nbsp;&nbsp;&nbsp;An overview of the campaign&rsquo;s metrics is shown whenever your mouse hovers over the Metric tab.</span></p>
<p><b>3.1.2 Add a Metric</b></p>
<p><span style="font-weight: 400;">Once in the Metric section, to add a new metric to the table:</span></p>
<ol>
<li style="font-weight: 400;"><span style="font-weight: 400;">Click the Add Metric button.</span></li>
<li style="font-weight: 400;"><span style="font-weight: 400;">Select a metric from the list. Selecting &ldquo;None&rdquo; effectively cancels your choice.</span></li>
<li style="font-weight: 400;"><span style="font-weight: 400;">Your metric is now listed. To add it to the table, click Update. To filter your metric, see </span><b>3.1.3</b><span style="font-weight: 400;">.</span></li>
</ol>
<p><b>3.1.3 Filter a metric </b></p>
<p><span style="font-weight: 400;"> &nbsp;&nbsp;&nbsp;You can filter a metric either on a date range, or by categorical data such as &nbsp;gender, age, income, and context. To filter an existing metric:</span></p>
<ol>
<li style="font-weight: 400;"><span style="font-weight: 400;">Select the metric from the table of added metrics.</span></li>
<li style="font-weight: 400;"><span style="font-weight: 400;">Alter filter constraints in the table by column. Some constraints may not work. See </span><b>FAQS</b><span style="font-weight: 400;">.</span></li>
<li style="font-weight: 400;"><span style="font-weight: 400;">Click Filter when all constraints are selected. Click Update to refresh the table.</span></li>
</ol>
<p><b>3.1.4 Delete a metric </b></p>
<p><span style="font-weight: 400;"> &nbsp;&nbsp;&nbsp;To delete a metric, navigate to the metric in your table, then click the Delete button on the metric.</span></p>
<p><b>3.1.5 Help with Metrics</b></p>
<p><span style="font-weight: 400;"> &nbsp;&nbsp;&nbsp;&nbsp;A helpful guide is provided in Dash</span> <span style="font-weight: 400;">within the Metric section. It covers section </span><b>3.1</b><span style="font-weight: 400;">, with additional explanations of each metric, and other helpful tips. To access this guide, when in the Metric section, click the ? button on the top right hand corner of the window.</span><b> &nbsp;</b></p>

<p><b>3.3 Compare Campaigns</b></p>
<p><b>3.3.1 Overview of Compare Campaigns </b></p>
<p><span style="font-weight: 400;"> &nbsp;&nbsp;&nbsp;An overview of how campaigns could be compared is shown whenever you hover your mouse over the Compare Campaigns button in the main application window.</span><b> &nbsp;</b></p>
<p><b>3.3.2 Compare Campaigns</b></p>
<p><span style="font-weight: 400;">Campaigns can be compared in two ways:</span></p>
<ol>
<li style="font-weight: 400;"><span style="font-weight: 400;">In tables, through side by side metric comparison</span></li>
<li style="font-weight: 400;"><span style="font-weight: 400;">Graphically, through the use of overlapping line graphs</span></li>
</ol>
<p><span style="font-weight: 400;"> &nbsp;&nbsp;First, load in a campaign to compare to your main campaign. Click Load Campaigns, then see </span><b>3.0.2</b><span style="font-weight: 400;">.</span></p>
<p><span style="font-weight: 400;">Once you have loaded a campaign:</span></p>
<ol>
<li style="font-weight: 400;"><span style="font-weight: 400;">To compare in a table, click the Table button. The table will load comparing all main metrics.</span></li>
<li style="font-weight: 400;"><span style="font-weight: 400;">To compare on a graph, click the Graph button. Graphs can be filtered by metrics. See </span><b>3.2</b><span style="font-weight: 400;">.</span></li>
</ol>
<p><span style="font-weight: 400;"> &nbsp;&nbsp;&nbsp;You can replace the second campaign with another by clicking the Load Campaign button at any time. When selecting the main campaign as the second campaign, see </span><b>FAQS</b><span style="font-weight: 400;">.</span></p>
<p><b>3.4 Settings</b></p>
<p><b>3.4.1 Calculating Bounce Rate </b></p>
<p><span style="font-weight: 400;">You can define how the bounce rate is calculated in all other features in this section. A visit by a user to an advertised page is a bounce when:</span></p>
<ol>
<li style="font-weight: 400;"><span style="font-weight: 400;">Time spent on the page is less than a required minimum to not be considered a bounce </span></li>
</ol>
<ol>
<li style="font-weight: 400;"><span style="font-weight: 400;">The number of pages viewed is less than a required minimum to not be considered a bounce</span></li>
</ol>
<p><span style="font-weight: 400;">It can be changed by selecting the checkbox of the option you wish to use.</span></p>
<p><b>3.4.2 Application Themes</b></p>
<p><span style="font-weight: 400;">To set the theme of Dash, click the Themes button in Settings. You can then either choose from: </span></p>
<ol>
<li style="font-weight: 400;"><span style="font-weight: 400;">Preset Themes</span></li>
<li style="font-weight: 400;"><span style="font-weight: 400;">A colour palette, where the selected colours are applied across the background, font and indicated accents throughout Dash.</span></li>
</ol>
